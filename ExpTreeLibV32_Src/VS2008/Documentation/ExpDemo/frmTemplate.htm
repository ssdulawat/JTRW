Summary

frmTemplate has an ExpTree Control in its' Left Panel and a ListView in its' Right Panel. The ExpTree shows a heirarchical view of Folders present on the host system. The root of the ExpTree display may be set to one of a number of Special Folders. The most commonly used of these are the Desktop or Computer (My Computer on some systems). Selecting a Node of this tree will cause the ListView to display the Content of the Selected Folder.

frmTemplate uses a single threaded method of populating the ListView. Working with Local Folders with a reasonable number of Files/Folders on a newer PC, this provides quite acceptable responsiveness. For applications designed to work with Remote Folders or very large Local Folders, frmThread will normally provide a better user experience as discussed here.

Capabilities

Simply displaying the structure and Content of Folders is not enough for a useful application. The real question is what the application can do with the displayed items. frmTemplate (as well as frmThread) supports the following capabilites:

  • Displaying the Folder content in most standard Views - Details, Large Icons, Tile, List, and Small Icons.
  • Dropping a File/Folder onto ExpTree and/or the ListView. This feature support the normal Windows Control-Key modifiers and Right Button Drag menu to instruct the receiving Control to Copy or Move the File(s)/Folders(s) to the Target Folder. The Drop Source may be itself or any window (including Windows Explorer) that provides at least FileDrop (CF_HDROP) DataFormatted information.
  • Dragging File(s)/Folder(s) from the ExpTree or the ListView. Normal Windows Control-Key and Right Button Drag operations are supported. The Drop Target may be itself or any window (including Windows Explorer) that accepts at least FileDrop (CF_HDROP) DataFormatted information.
  • Right-Click on a File/Folder to display the same ContextMenu that Windows Explorer would display to perform the same operations. File/Folder Rename support included.
  • Edit of the First Column of the ListView -- if and only if the first Column is the File/Folder name.
  • Automatic Update of the ListView when any changes are made by any process to the displayed Folder or its' Content.
  • Column-Click Sorting (with Sort Glyph) of the ListView Columns when displayed in Details View.
  • Selected Item(s) Delete in response to the Delete Key.
  • Double-Click on a File to "Open" it.

Modifying the ListView for an Application

frmTemplate is a base, worked example of how to implement these capabilites in a ListView. An application will probably have diferent or additional requirements. The Cardinal Rule when modifying the ListView is that each ListViewItem must have the appropriate CShItem instance in its' .Tag Property. The underlying implication is that each ListViewItem represents a single, unique File or Folder, all within one single Folder (e.g. the Desktop or Computer or C:\Users\username\Documents\Fax). Additionally, the ListView itself must have its' .Tag Property set to the CShItem representing the single Folder that contains all ListViewItems and their associated CShItems.

A second rule is that the File/Directory Name should be in the first (column 0) column. This is not quite as inflexible, but Rename and, probably, some other functions depend on it. Do not violate the rule without very serious understanding of how the ListView is used by the Form.

See frmTemplate.MakeLVItem for how to populate the ListView.
